## Бесплатное продвижение сайта

Основой бесплатного продвижения сайта является внутреннее SEO или, другими словами, оптимизация контента на вебсайте под запросы поисковых систем, т.к. она способна существенно (в разы) снизить стоимости продвижения.

Это связано с тем, что при хорошей оптимизации текстов под запросы поисковых систем уже не нужно будет закупать огромное количество внешних (обратных) ссылок. Можно будет существенно на этом сэкономить.

https://slides.com/janusnicon/insightseo

## Gulp-useref

https://github.com/jonkemp/gulp-useref

Gulp-useref объединяет любое количество CSS и JavaScript файлов в один с помощью комментариев:

```
начальный «<!—build:» и конечный «<!—endbuild—>». 
```

Синтаксис:

```html
<!-- build:<type> <path> -->
... HTML разметка, скрипты/ ссылки.
<!-- endbuild -->
```

- type - может принимать значения js, css или remove. Лучше всего устанавливать типы в соответствии с файлами, которые хотите объединить. Если установить тип remove, то Gulp удалит весь блок между комментариев.

- path — путь к выходному файлу. Нам надо, чтобы финальный файл JavaScript лежал в папке js с именем main.min.js. 

Следовательно код будет следующий:

```html
<!--build:js js/main.min.js -->
<script src="js/lib/a-library.js"></script>
<script src="js/lib/another-library.js"></script>
<script src="js/main.js"></script>
<!-- endbuild -->
```

## Создание задачи useref


```js
// функция useref.assets() должна вызываться перед gulp.src.
gulp.task('useref', function(){
var assets = useref.assets();

return gulp.src('app/*.html')
.pipe(assets)
.pipe(assets.restore())
.pipe(useref())
.pipe(gulp.dest('dist'))
});
```

Если запустить useref, Gulp пробежится по трем скриптам и соединит их в один dist/js/main.min.js.

Однако сам файл еще не минифицирован. 


## Минификация JS файлов

плагин gulp-uglify используется для минификации JS файлов.


gulp-uglify — JavaScript компрессор.
https://github.com/terinjokes/gulp-uglify


```js

$ npm install gulp-uglify --save-dev
 
// другие подключения...
var uglify = require('gulp-uglify');

gulp.task('useref', function(){
var assets = useref.assets();

return gulp.src('app/*.html')
.pipe(assets)
.pipe(uglify())
.pipe(assets.restore())
.pipe(useref())
.pipe(gulp.dest('dist'))
});

```

Теперь при запуске задачи useref , Gulp будет автоматически минифицировать файл `main.min.js`. Наш плагин gulp-useref автоматически объединяет скрипты между комментариев 

```
«<!—build:» и «<!—endbuild—>» 
```
в один файл и помещает весь код в `js/main.min.js`.

## соединить CSS файлы

Таким же способом мы можем соединить CSS файлы (если их у вас будет больше одного). Шаги все те же самые, добавляем комментарий.


## build:css
```html
<html>
<head>
    <!-- build:css css/combined.css -->
    <link href="css/one.css" rel="stylesheet">
    <link href="css/two.css" rel="stylesheet">
    <!-- endbuild -->
</head>
```

## HTML:

```html
<html>
<head>
    <link rel="stylesheet" href="css/combined.css"/>
</head>
```

Так как мы используем CSS, то задача useref слегка изменится. Нам не нужно запускать функцию uglify() для CSS файлов, она все испортит. 

Нужно удостовериться, что uglify() запускается только для JS файлов, в этом нам поможет gulp-if.

```js
$ npm install gulp-if --save-dev

// другие подключения...
var gulpIf = require('gulp-if');
gulp.task('useref', function(){
var assets = useref.assets();

return gulp.src('app/*.html')
.pipe(assets)

// Если JS то запускаем uglify()

.pipe(gulpIf('*.js', uglify()))

.pipe(assets.restore())
.pipe(useref())

.pipe(gulp.dest('dist'))

});
```

## Минифицируем только CSS файлы

Чтобы минифицировать CSS файлы нам потребуется плагин gulp-minify-css.

```js

$ npm install gulp-minify-css

var minifyCSS = require('gulp-minify-css');

gulp.task('useref', function(){

var assets = useref.assets();

return gulp.src('app/*.html')

.pipe(assets)

// Минифицируем только CSS файлы

.pipe(gulpIf('*.css', minifyCSS()))

// Uglifies only if it's a Javascript file
.pipe(gulpIf('*.js', uglify()))
.pipe(assets.restore())
.pipe(useref())
.pipe(gulp.dest('dist'))

});
```
Теперь на выходе мы получаем по одному оптимизированному CSS и JavaScript файлу при любом запуске задачи useref.

## Автоматическое удаление сгенерированных файлов

Так как мы генерируем наши файлы автоматически, то нам необходимо быть уверенными, что те файлы, которые нам больше не нужны нигде не завалялись. Этот процесс называется очистка. Для очистки воспользуемся del.

```js
npm install del --save-dev

var del = require('del');
```
Функция del принимает массив подстановок Node, в котором содержатся папки на удаление. Настройка del почти такая же, как и настройка первой задачей, которую мы создавали:
```js
gulp.task('clean', function() {
del('dist');
})
```

Теперь Gulp удалит папку dist при любом запуске задачи gulp clean. 
создать отдельную задачу для удаления всех файлов в папке dist кроме изображений.

```js
gulp.task('clean:dist', function(callback){
del(['dist/**/*', '!dist/images', '!dist/images/**/*'], callback)
});
```
Прежде всего, мы удаляем все файлы в папке `dist/**/*`. Так как мы не хотим удалять `dist/images`, то нужно применить символ «!». Также нам нельзя удалять любые файлы внутри папки `dist/images`, так что нам нужна третья подстановка. И наконец, нам нужно знать, когда задача clean:dist выполнилась, для этого необходимо добавить аргумент в функцию. По умолчанию при использовании задачи clean удаляется кэш изображений, значит, задаче images придется обрабатывать их заново.

```js
gulp.task('clean', function(callback) {

del('dist');
return cache.clearAll(callback);
})
```

## Внешняя SEO оптимизация

При полностью бесплатном продвижении сайта нужно понимать, что без тематических внешних обратных ссылок, закупленных в достаточно большом количестве, продвинуться по высокочастотным и высококонкурентным поисковым запросам у вас не получится.

## Низкочастотные запросы
Низкочастотные запросы — это запросы в поисковой системе, имеющие узкую направленность. В SEO различают три вида запросов: высокочастотные (ВЧ), среднечастотные (СЧ) и низкочастотные (НЧ). В своей формулировке низкочастотный запрос более точно отражает требование пользователя (например, ремонт электротехники). Такой запрос представляет собой несколько слов, фраз, названия определённых марок товара. Частота его показа в среднем может составлять около 1000 просмотров в месяц.


## Использование

Сайты с низкочастотными запросами в основном используются:

    компаниями с множеством разнообразных товаров и услуг, не заменяющими друг друга;
    начинающими компаниями, желающими сэкономить бюджет;
    компаниями, занимающими успешные места по высокочастотным запросам и желающими увеличить количество посетителей за счёт НЧ запросов;
    компаниями, чья целевая аудитория состоит из посетителей, твёрдо сформулировавших свои потребности и из специалистов.

## Технология

Оптимизация сайта для низкочастотных запросов предполагает продвижение множества страниц с запросами, задаваемыми наиболее часто. В большинстве случаев действует принцип «один запрос — одна страница». Зачастую низкочастотный трафик достигается при помощи несложной внутренней оптимизации. В условиях конкуренции в определённой области используется внешняя оптимизация (покупка нескольких ссылок с НЧ текстами). Для того чтобы расположить сайт на первых местах не обязательно делать «закупки» больших объёмов. В размещении низкочастотных запросов основным критерием является общая оценка страницы, её качества. Из-за меньшей конкуренции для продвижения подходят страницы не только первого уровня вложенности, имеющие даже небольшой PR (PageRank). Также важными являются: ТИЦ сайта с определённой страницей, характеристика ссылок относительно их фильтрации, численность возможных и текущих внешних ссылок на покупаемой странице.

## Бесплатные внешние ссылки

Бесплатные обратные ссылки являются очень существенным элементом общего бесплатного продвижения сайта.
Дело в том, что даже по гиперссылкам с профилей вебсайтов (и форумов тоже) на ваш ресурс передается определенный вес, который при надлежащей внутренней перелинковке будет равномерно распределяться по всему ресурсу и благополучно достигнет страниц со статьями, которые вы, собственно, и продвигаете в топ поисковой выдаче по тем или иным низкочастотным запросам.
Это основная функция внутренней перелинковки страниц вебсайта, которая является очень важной при бесплатном продвижении, когда вся основная ссылочная масса, добытая на трастовых ресурсах, будет приходиться на главную страницу проекта. И именно внутренняя перелинковка страниц позволит статическому весу, передаваемому с трастовых сайтов, благополучно перетечь на страницы со статьями.

Бесплатные внешние ссылки позволят вам нагнать статический вес на вебсайт, а грамотная внутренняя перелинковка позволит этому статическому весу благополучно распределиться по всему ресурсу и перетечь на страницы со статьями, которые вы продвигаете бесплатными методами. Но не забывайте, что статический вес передается и со страниц вашего собственного ресурса, поэтому значимость правильной внутренней перелинковки страниц сайта еще более повышается.

## Существует два типа ссылок: анкорные и безанкорные.

Анкорные ссылки – это те, у которых ссылка спрятана под текстом, а при нажатии на него вы попадаете на другой сайт. Например «блог об интернет-маркетинге» — это ссылка с анкором. Если кликните по ней, попадете на главную блога «Нетологии».

## Безанкорная ссылка
«Голая» ссылка на блог https://example.com/blog — это безанкорная ссылка. К безанкорным ссылкам также относятся ссылки, которые спрятаны под текстом не несущим смысловой нагрузки^ «здесь, тут, сюда, подробнее».

Анкорные ссылки помогают продвигать страницы сайта по запросам, а безанкорные ссылки улучшают трастовость (показатель доверия поисковых систем к сайту) и авторитетность домена. В результате поисковые роботы охотнее индексируют ваш сайт и быстрее продвигают его вверх в поисковой выдаче.

Так как ссылки прямым образом влияют на ранжирование, хорошие сеошники используют только естественные обратные ссылки с авторитетных источников в глазах поисковиков. К сожалению, качественные бэклинки получить не так-то дешево, а если закупать плохие ссылки, то сайт залетит под санкции Google и Яндекса.

Сегодня очень важно регулярно проводить анализ ссылочной массы своего сайта и сайтов конкурентов. Систематический поиск и определение полезных линков у сайтов, которые находятся в ТОПе — задача №1 для современного seo-специалиста. Это поможет в продвижении сайта внешними ссылками.

## Вебмастер Google
https://www.google.com/webmasters/#?modal_active=none

Если вы имеете доступы в Google Search Console, вы можете узнать о всех ссылках на ваш ресурс, которые нашел Google. Для этого переходим в раздел Ссылки на ваш сайт:

В этом отчете можете видеть суммарное количество ссылок со всех сайтов, а также посмотреть подробно, с каких сайтов они ведут. Для этого необходимо перейти по ссылке Дополнительно:

Здесь вам будет предоставлена информация по каждому доменному имени (сколько раз он ссылается на ваш сайт и на какое количество страниц).

Чтобы увидеть самые популярные страницы, на которые ведут ссылки, необходимо перейти в отчет Все страницы, на которые ссылаются:

Тексты ссылок можно увидеть в отчете Ссылки на ваши данные

Все таблицы можно легко экспортировать, для подробного изучения и дальнейшей работы с этими данными.
